== アプリケーションの保護ポリシーの変更

アプリケーションに関連付けられた保護ポリシーを変更して別のバケットまたはセカンダリレプリケーションターゲットを使用する場合、保護ポリシーが失敗する可能性があります。

*回避策*：アプリケーションを削除して再度追加します。



== 3-2-1保護ポリシー設定の制限

オブジェクトストレージのバックアップがセカンダリストレージへのレプリケーションと同時に行われない3-2-1保護ポリシーを作成すると、アプリケーションを保護するときに保護ポリシーがハングします。

*回避策*：アプリケーションを削除して再度追加し、別の保護ポリシーで保護します。
